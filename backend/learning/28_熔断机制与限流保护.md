### 28.2 熔断器模式原理

```
熔断器状态机：

            ┌─────────┐
            │  CLOSED │ ←─ 正常状态，请求通过
            └────┬────┘
                 │
         失败超过阈值
                 │
                 ↓
            ┌─────────┐
            │  OPEN   │ ←─ 熔断状态，快速失败
            └────┬────┘
                 │
          等待恢复时间
                 │
                 ↓
            ┌─────────┐
            │HALF-OPEN│ ←─ 半开状态，尝试少量请求
            └────┬────┘
                 │
    ┌────────────┴────────────┐
    │                         │
    成功                    失败
    │                         │
    ↓                         ↓
┌─────────┐              ┌─────────┐
│  CLOSED │              │  OPEN   │
└─────────┘              └─────────┘
```

### 28.2 TokenBucketRateLimiter 令牌桶限流器

```python
class TokenBucketRateLimiter:
    """
    令牌桶限流器
    
    原理：
    - 桶有固定容量
    - 按固定速率向桶中添加令牌
    - 请求需要获取令牌才能通过
    - 没有令牌则拒绝
    
    Redis 实现（分布式限流）：
    - 使用 Redis 计数器
    - 设置过期时间作为时间窗口
    """
    
    def __init__(self, rate=60, capacity=100, window=60):
        """
        初始化限流器
        
        rate: 令牌生成速率（个/窗口）
        capacity: 桶容量（最大突发量）
        window: 时间窗口（秒）
        """
        self.rate = rate
        self.capacity = capacity
        self.window = window
        self.redis = redis_manager
    
    async def acquire(self, key):
        """
        获取令牌
        
        key: 限流维度（如 user_id, enterprise_id）
        """
        redis_key = f"ratelimit:{key}"
        
        try:
            await self.redis.connect()
            
            current = await self.redis.get(redis_key)
            count = int(current.decode('utf-8')) if current else 0
            
            if count >= self.rate:
                raise RateLimitExceeded(f"Rate limit exceeded: {self.rate} per {self.window}s")
            
            if count == 0:
                await self.redis.set(redis_key, 1, ex=self.window)
            else:
                await self.redis.incr(redis_key)
            
            return True
        except RateLimitExceeded:
            raise
        except Exception as e:
            print(f"Rate limiter error: {e}")
            return True  # 错误时降级处理
```

### 28.3 CircuitBreaker 熔断器

```python
class CircuitBreaker:
    """熔断器"""
    
    def __init__(self, failure_threshold=5, recovery_timeout=60, half_open_max_calls=3):
        self.failure_threshold = failure_threshold  # 失败阈值
        self.recovery_timeout = recovery_timeout      # 恢复等待时间
        self.half_open_max_calls = half_open_max_calls  # 半开状态最大请求数
        
        self._state = CircuitState.CLOSED
        self._failure_count = 0
        self._last_failure_time = None
        self._half_open_calls = 0
        self._last_state_change = datetime.utcnow()
    
    @property
    def state(self):
        """获取当前状态（自动处理 OPEN → HALF_OPEN 转换）"""
        if self._state == CircuitState.OPEN:
            if self._last_failure_time:
                elapsed = (datetime.utcnow() - self._last_failure_time).total_seconds()
                if elapsed >= self.recovery_timeout:
                    self._state = CircuitState.HALF_OPEN
                    self._half_open_calls = 0
                    self._last_state_change = datetime.utcnow()
        return self._state
    
    def record_success(self):
        """记录成功"""
        if self._state == CircuitState.HALF_OPEN:
            self._half_open_calls += 1
            if self._half_open_calls >= self.half_open_max_calls:
                self._state = CircuitState.CLOSED
                self._failure_count = 0
                self._last_state_change = datetime.utcnow()
        else:
            self._failure_count = 0
    
    def record_failure(self):
        """记录失败"""
        self._failure_count += 1
        self._last_failure_time = datetime.utcnow()
        
        if self._state == CircuitState.HALF_OPEN:
            self._state = CircuitState.OPEN
            self._last_state_change = datetime.utcnow()
        elif self._failure_count >= self.failure_threshold:
            self._state = CircuitState.OPEN
            self._last_state_change = datetime.utcnow()
    
    def can_execute(self):
        """检查是否可以执行"""
        if self.state == CircuitState.CLOSED:
            return True
        elif self.state == CircuitState.HALF_OPEN:
            return self._half_open_calls < self.half_open_max_calls
        else:
            return False
```

### 28.2 LLM 熔断器使用

```python
class LLMCircuitBreaker:
    """LLM API 专用熔断器"""
    
    DEFAULT_FALLBACKS = [
        FallbackResponse(
            message="抱歉，我现在遇到了一些技术问题，无法立即回答您的问题。",
            suggested_action="您可以稍后再试，或者选择转人工客服获取帮助。",
            can_human_takeover=True
        ),
        FallbackResponse(
            message="系统繁忙，请稍等片刻。",
            suggested_action="如果问题紧急，请联系人工客服。",
            can_human_takeover=True
        ),
    ]
    
    def __init__(self, failure_threshold=5, recovery_timeout=60):
        self.circuit_breaker = CircuitBreaker(
            failure_threshold=failure_threshold,
            recovery_timeout=recovery_timeout
        )
        self.fallbacks = self.DEFAULT_FALLBACKS
        self._fallback_index = 0
    
    def get_fallback(self):
        """获取下一个降级响应（轮询）"""
        fallback = self.fallbacks[self._fallback_index]
        self._fallback_index = (self._fallback_index + 1) % len(self.fallbacks)
        return fallback
    
    async def execute(self, func, *args, **kwargs):
        """
        执行函数，带熔断器保护
        
        流程：
        1. 检查熔断器状态
        2. 如果 OPEN → 返回降级响应
        3. 否则执行函数
        4. 记录成功/失败
        5. 如果失败且熔断器变为 OPEN → 返回降级
        """
        if not self.circuit_breaker.can_execute():
            return self.get_fallback()
        
        try:
            result = await func(*args, **kwargs)
            self.circuit_breaker.record_success()
            return result
        except Exception as e:
            self.circuit_breaker.record_failure()
            
            if self.circuit_breaker.state == CircuitState.OPEN:
                return self.get_fallback()
            
            raise
```

### 28.3 完整的保护链

```python
async def protected_chat(user_id, enterprise_id, query):
    """
    完整的保护链：
    1. 限流检查
    2. 熔断器检查
    3. 实际调用
    """
    
    # 1. 限流
    limiter = get_llm_rate_limiter()
    try:
        await limiter.acquire(f"user:{user_id}")
        await limiter.acquire(f"enterprise:{enterprise_id}")
    except RateLimitExceeded:
        return {
            "type": "error",
            "message": "请求过于频繁，请稍后再试",
            "retry_after": 60
        }
    
    # 2. 熔断器
    breaker = get_llm_circuit_breaker()
    
    # 3. 调用
    result = await breaker.execute(
        llm.generate,
        query
    )
    
    # 4. 返回结果
    if hasattr(result, 'to_dict'):
        return {
            "type": "fallback",
            **result.to_dict()
        }
    else:
        return {
            "type": "success",
            "answer": result
        }
```

## 5. 工程化思维要点

### 28.1 限流维度设计

```
限流维度选择：

用户级（最细粒度）：
- Key: user:{user_id}
- 防止单个用户刷量
- 适合用户-facing API

企业级（租户隔离）：
- Key: enterprise:{enterprise_id}
- 防止某个企业占用太多资源
- 适合 SaaS 系统

接口级（全局保护）：
- Key: endpoint:{endpoint}
- 保护某个接口不被打爆
- 适合关键接口

综合使用：
- 同时应用多个维度
- 取最严格的限制
```
