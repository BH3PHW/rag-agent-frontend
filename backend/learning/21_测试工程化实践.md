## 21.1 测试策略全景

### 21.1.1 为什么测试至关重要？

**产品化思维**：测试是产品质量的保障

```
测试对产品的价值：
┌────────────────────────────────────────────────────────────┐
│  1. 质量保障 - 确保产品功能符合预期                     │
│  2. 风险降低 - 提前发现bug，减少线上事故                 │
│  3. 重构信心 - 有测试保障，敢重构代码                     │
│  4. 文档作用 - 测试用例是最好的文档                       │
│  5. 用户信任 - 稳定的产品赢得用户信任                     │
└────────────────────────────────────────────────────────────┘
```

### 21.1.2 测试金字塔

**工程化最佳实践**：合理的测试分层

```
测试金字塔（Test Pyramid）：
┌─────────────────────────────────────────────────────────────┐
│  E2E测试（端到端）  ↑  数量少，覆盖核心流程，速度慢     │
│  ──────────────────────────────────────────────────────  │
│  集成测试（Integration）│  中等数量，测试组件交互         │
│  ──────────────────────────────────────────────────────  │
│  单元测试（Unit）       ↓  数量多，覆盖代码细节，速度快   │
└─────────────────────────────────────────────────────────────┘
```

**不同测试类型的特点**：

| 测试类型 | 目标 | 数量 | 速度 | 维护成本 |
|---------|------|------|------|---------|
| 单元测试 | 函数/类级别 | 多 | 快 | 低 |
| 集成测试 | 服务/模块交互 | 中 | 中 | 中 |
| E2E测试 | 完整业务流程 | 少 | 慢 | 高 |

# 文件：backend/tests/test_chat_service.py
import pytest
from unittest.mock import Mock, AsyncMock, patch
from datetime import datetime

# 被测试的函数
def calculate_token_cost(input_tokens: int, output_tokens: int) -> float:
    """计算Token成本"""
    input_cost = input_tokens * 0.001
    output_cost = output_tokens * 0.002
    return input_cost + output_cost

# 单元测试
class TestCalculateTokenCost:
    """Token成本计算测试"""
    
    def test_normal_case(self):
        """正常情况测试"""
        result = calculate_token_cost(100, 200)
        assert result == 0.5  # 100*0.001 + 200*0.002 = 0.1 + 0.4 = 0.5
    
    def test_zero_input(self):
        """零输入边界测试"""
        result = calculate_token_cost(0, 100)
        assert result == 0.2
    
    def test_zero_output(self):
        """零输出边界测试"""
        result = calculate_token_cost(100, 0)
        assert result == 0.1
    
    def test_both_zero(self):
        """全零边界测试"""
        result = calculate_token_cost(0, 0)
        assert result == 0.0
    
    @pytest.mark.parametrize("input_tokens, output_tokens, expected", [
        (10, 20, 0.05),
        (1000, 500, 2.0),
        (500, 1000, 2.5),
    ])
    def test_parameterized(self, input_tokens, output_tokens, expected):
        """参数化测试 - 工程化：减少重复代码"""
        result = calculate_token_cost(input_tokens, output_tokens)
        assert result == expected
```

import pytest
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
from common.database import Base

# 工程化：测试数据库配置
TEST_DATABASE_URL = "sqlite:///./test.db"

@pytest.fixture(scope="function")
def db_session():
    """
    数据库会话fixture
    
    工程化：
    - 每个测试使用独立的数据库
    - 测试后自动回滚
    - 隔离测试之间的影响

    engine = create_engine(TEST_DATABASE_URL, connect_args={"check_same_thread": False})

    session = TestingSessionLocal()
    try:
        yield session
    finally:
        session.rollback()
        session.close()
        # 清理表
        Base.metadata.drop_all(bind=engine)

### 21.5.2 Fixture 与数据工厂

# 文件：backend/tests/factories.py
import factory
from datetime import datetime
from chat_service.models import ChatSession, ChatMessage

class ChatSessionFactory(factory.Factory):
    """会话工厂 - 工程化：快速生成测试数据"""
    
    class Meta:
        model = ChatSession
    
    id = factory.Sequence(lambda n: f"session-{n:04d}")
    user_id = factory.Sequence(lambda n: f"user-{n:04d}")
    enterprise_id = "test-enterprise-001"
    created_at = factory.LazyFunction(datetime.now)
    updated_at = factory.LazyFunction(datetime.now)

class ChatMessageFactory(factory.Factory):
    """消息工厂"""
    
    class Meta:
        model = ChatMessage
    
    id = factory.Sequence(lambda n: f"msg-{n:04d}")
    session_id = factory.SubFactory(ChatSessionFactory)
    role = "user"
    content = factory.Faker("sentence")
    created_at = factory.LazyFunction(datetime.now)

# 使用示例
def test_with_factory(db_session):
    """使用工厂生成测试数据"""
    session = ChatSessionFactory.create()
    messages = ChatMessageFactory.create_batch(5, session_id=session.id)
    
    assert len(messages) == 5
    assert messages[0].session_id == session.id
```

### 21.5.3 Test Doubles 模式

| 类型 | 用途 |

from unittest.mock import Mock, MagicMock, spy

## 21.9 本章小结

### 核心测试概念

| 概念 | 工程化价值 | 产品化价值 |
|------|-----------|-----------|
| **单元测试** | 代码质量，快速反馈 | 基础功能稳定 |
| **集成测试** | 组件交互正确 | 业务流程正常 |
| **E2E测试** | 端到端验证 | 真实用户体验 |
| **覆盖率** | 代码质量指标 | 质量保障信心 |
| **压力测试** | 性能优化 | 用户体验保障 |

### 工程化思维要点

1. **测试金字塔** - 合理的测试分层
2. **测试隔离** - 测试之间互不影响
3. **快速反馈** - 单元测试要快
4. **可维护测试** - 测试代码也是生产代码
5. **持续集成** - 每次提交都运行测试

### 产品化思维要点

1. **核心流程** - 重点测试核心业务流程
2. **用户视角** - E2E测试模拟真实用户
3. **性能保障** - 压力测试发现瓶颈
4. **质量信心** - 高覆盖率给发布信心
5. **风险降低** - 提前发现问题，减少线上事故
