## 1. 查询预处理简介

### 30.1 为什么需要预处理

```
用户输入的问题：
- 有拼写错误
- 有语法问题
- 标点符号不规范
- 有重复内容
- 有特殊字符

问题：
1. 拼写错误影响检索质量
2. 不规范输入影响语义理解
3. 降低用户体验

解决方案：
在进入 RAG 流程前，先做预处理！
```

### 30.2 预处理流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      查询预处理流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  用户原始查询 → [步骤1] 去空白 → [步骤2] 标点规范化 → [步骤3]   │
│  拼写纠错 → [步骤4] 去特殊字符 → 规范化查询                     │
│                                                                 │

```

    - 同音字错误
    - 键盘相邻错误
    - 常见错别字

        # 拼音错误映射
        self.PINYIN_ERRORS = {
            "蓝": "南", "啦": "哪", "里": "你", "了": "呢",

            'c': 'xdfv', 'v': 'cfgb', 'b': 'vghn',
            'n': 'bhjm', 'm': 'njk'

### 30.2 纠错方法实现

```python
    def correct(self, text):
        """
        纠错主方法
        
        返回：
        (corrected_text, corrections_list)
        """
        corrections = []
        corrected = text
        
        # 1. 词组级别纠错（优先）
        for wrong, right in self.COMMON_TYPOS.items():
            if wrong in corrected:
                corrections.append({
                    "type": "typo",
                    "original": wrong,
                    "corrected": right,
                    "position": corrected.find(wrong)
                })
                corrected = corrected.replace(wrong, right)
        
        # 2. 单字级别纠错（拼音相似）
        for wrong, right in self.PINYIN_ERRORS.items():
            if wrong in corrected:
                corrections.append({
                    "type": "pinyin",
                    "original": wrong,
                    "corrected": right,
                    "position": corrected.find(wrong)
                })
                corrected = corrected.replace(wrong, right, 1)
        
        # 3. 重复字符纠正
        matches = self.typo_patterns.findall(corrected)
        for match in matches:
            if len(match) >= 2:
                corrections.append({
                    "type": "repeat",
                    "original": match * 2,
                    "corrected": match,
                    "position": corrected.find(match * 2)
                })
        
        return corrected, corrections
```

### 30.3 QueryPreprocessor 查询预处理器

```python
class QueryPreprocessor:
    """
    查询预处理器
    
    功能：
    - 拼写纠错
    - 空白处理
    - 标点规范化
    - 特殊字符过滤
    """
    
    def __init__(self, enable_correction=True):
        self.spell_corrector = SpellingCorrector()
        self.enable_correction = enable_correction
        
        # 标点规范化映射
        self.punctuation_map = {
            '，': ',',
            '。': '.',
            '！': '!',
            '？': '?',
            '；': ';',
            '：': ':',
            '（': '(',
            '）': ')',
            '【': '[',
            '】': ']',
            '《': '<',
            '》': '>',
        }
```

### 30.4 预处理流程实现

```python
    def preprocess(self, query):
        """
        完整预处理流程
        
        返回：
        {
            "original": 原始查询,
            "corrected": 纠错后,
            "corrections": 纠错列表,
            "normalized": 最终规范化查询,
            "warnings": 警告信息
        }
        """
        result = {
            "original": query,
            "corrected": query,
            "corrections": [],
            "normalized": query,
            "warnings": []
        }
        
        # 步骤1：去除首尾空白
        processed = query.strip()
        
        # 步骤2：标点规范化
        for old, new in self.punctuation_map.items():
            if old in processed:
                processed = processed.replace(old, new)
        
        # 步骤3：多余空格处理
        processed = re.sub(r'\s+', ' ', processed)
        
        # 步骤4：拼写纠错
        if self.enable_correction:
            corrected, corrections = self.spell_corrector.correct(processed)
            result["corrected"] = corrected
            result["corrections"] = corrections
            
            if corrections:
                processed = corrected
                result["warnings"].append(
                    f"已自动纠正 {len(corrections)} 处拼写错误"
                )
        
        # 步骤5：去除控制字符
        processed = ''.join(
            char for char in processed
            if ord(char) >= 32 or char in '\n\t'
        )
        
        result["normalized"] = processed
        return result
    
    def preprocess_for_retrieval(self, query):
        """
        为检索做预处理（简化版）
        只返回规范化的查询文本
        """
        result = self.preprocess(query)
        return result["normalized"]
```

async def rag_with_preprocessing(query, enterprise_id):

    带预处理的 RAG 流程

    # 使用预处理后的查询进行检索
    retrieval_result = await retrieve(
        query=preprocessed['normalized'],
        enterprise_id=enterprise_id
    )
    
    # 返回结果，同时返回预处理信息
    return {
        "retrieval": retrieval_result,
        "preprocessing": preprocessed
    }

"检测到可能的拼写错误，是否纠正？"

## 6. 最佳实践

### 30.1 纠错策略建议

```
何时启用纠错：
✅ 启用：用户输入短（10-20字）
✅ 启用：常见查询场景
❌ 谨慎：专业术语（可能误纠）
❌ 谨慎：专有名词（人名、产品名）

可配置化：
- 按企业配置是否启用
- 按场景配置严格度
- 提供白名单（不纠某些词）
```

### 30.2 多级预处理

```
更高级的预处理管道：

第1级：基础清洗
- 去空白
- 去特殊字符
- 标点规范化

第2级：纠错
- 拼写纠错
- 语法纠错
- 术语归一化

第3级：增强
- 查询扩展
- 关键词提取
- 重写优化

第4级：过滤
- 敏感词过滤
- 无意义查询过滤
```
