# RAG Agent 开发学习教程

> 🎓 **目标读者**：学习 RAG Agent 开发的学生和开发者  
> 📚 **学习方式**：从实际项目中学习  
> 💡 **核心理念**：通过代码理解理论 + **工程化思维** + **产品化思维**

---

## 课程概览

本教程基于一个完整的**RAG智能客服系统**项目，不仅学习技术实现，更注重培养工程化和产品化思维。

### 学习路线

```
第1阶段：打基础
├─ 架构篇 → 理解系统设计思想
├─ 工程化篇 → 掌握生产开发技能
└─ 快速上手 → 运行项目看效果

第2阶段：学核心
├─ RAG篇 → 掌握检索增强生成
└─ Agent篇 → 理解智能体决策

第3阶段：实践
├─ 开发实践 → 动手实现功能
└─ 数据处理 → 保障数据质量
```

## 💡 两大核心理念

### 🛠️ 工程化思维
> 写出能在生产环境稳定运行的代码

| 工程化要素 | 体现 |
|----------|------|
| 代码质量 | 分层架构、模块化设计、可复用 |
| 可维护性 | 清晰的代码结构、完善的日志 |
| 可观测性 | 监控指标、质量评估、健康检查 |
| 性能优化 | 缓存策略、连接池、限流保护 |
| 安全防护 | 数据脱敏、输入验证、权限控制 |

### 🎯 产品化思维
> 从用户视角设计和优化产品

| 产品化要素 | 体现 |
|----------|------|
| 用户体验 | 流式响应打字机效果、快速反馈 |
| 数据质量 | 清洗管道、质量评估、保障回答准确 |
| 成本控制 | 混合检索、缓存策略、节省LLM费用 |
| 高可用 | 健康检查、自动重启、多实例部署 |
| 安全可靠 | 防提示词注入、内容安全、限流保护 |

## 🛠️ 技术栈

### 后端技术栈
- **框架**：FastAPI (高性能异步框架)
- **数据库**：PostgreSQL (关系型) + ChromaDB (向量)
- **缓存**：Redis
- **大模型**：通义千问 (Qwen)
- **部署**：Docker + Docker Compose

### 前端技术栈
- **框架**：React 18 + TypeScript
- **状态管理**：Zustand
- **样式**：Tailwind CSS
- **构建**：Vite

## 🚀 快速开始

### 第一步：环境准备
```bash
# 确保安装了以下软件
Python 3.10+
Node.js 18+
PostgreSQL 14+
Redis 6+
Docker (可选)
```

### 第二步：项目运行
```bash
# 1. 克隆项目（如果还没）
cd /workspace

# 2. 启动后端服务（按顺序）
cd backend/api-gateway
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# 3. 在新终端启动其他服务
# user-service (8001)
# chat-service (8002)
# knowledge-service (8003)

# 4. 启动前端
cd /workspace/frontend
npm install
cd apps/consumer
npm run dev  # 端口3001
```

详细步骤请参考：[23_快速上手.md](./23_快速上手.md)

## 💡 学习建议

### 学习方法
1. **先理解为什么，再看怎么做**：每个决策都有原因
2. **动手运行调试**：修改代码观察效果
3. **按顺序学习**：从架构到细节，循序渐进
4. **做笔记**：记录关键概念和自己的理解
5. **思考工程化/产品化**：每个功能都想为什么这样实现

### 阅读策略
- **架构文档**：理解为什么这样设计（权衡、选择）
- **工程化文档**：重点看设计模式和最佳实践
- **代码注释**：重点看关键逻辑的注释
- **API文档**：FastAPI自动生成的 `/docs` 很有帮助
- **实际调试**：通过断点和日志理解执行流程

## 🤔 常见问题

**Q: 我是完全的新手，能学会吗？**
A: 可以！我们从基础概念讲起，配合具体代码示例，循序渐进。重点是理解设计思想。

**Q: 需要什么前置知识？**
A: 基础的Python编程，了解HTTP和REST API概念即可。如果有Docker基础更好。

**Q: 大模型API需要申请吗？**
A: 项目使用模拟数据可以先学习，需要时再申请真实API。

**Q: 工程化和产品化具体是指什么？**
A: 工程化关注代码质量、可维护性、可观测性；产品化关注用户体验、成本、质量、稳定。我们的课程会详细讲解！

**Q: 学习过程中遇到问题怎么办？**
A: 先看快速上手文档，里面有常见问题排查方法。
