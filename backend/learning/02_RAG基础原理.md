### 2.2.2 向量检索原理

**为什么用向量？**

```
┌─────────────────────────────────────────────────────────────────────┐
│                       词向量 vs 语义向量                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  传统关键词匹配（词向量）：                                          │
│  ┌─────────────────────────────────────────────────────────┐        │
│  │                                                          │        │
│  │  用户输入："取消订单"                                    │        │
│  │                                                          │        │
│  │  文档A："如何取消订单"      ← 包含关键词，匹配成功 ✓     │        │
│  │  文档B："订单怎么退掉"      ← 不包含关键词，匹配失败 ✗   │        │
│  │  文档C："不想买了怎么办"    ← 语义相同，匹配失败 ✗       │        │
│  │                                                          │        │
│  │  问题：意思相同但表述不同的内容找不到                     │        │
│  │                                                          │        │
│  └─────────────────────────────────────────────────────────┘        │
│                                                                       │
├───────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  语义搜索（向量）：                                                   │
│  ┌─────────────────────────────────────────────────────────┐        │
│  │                                                          │        │
│  │  用户输入："取消订单"                                    │        │
│  │    ↓ 向量化                                              │        │
│  │  向量A：[0.2, 0.8, 0.1, ...]                            │        │
│  │                                                          │        │
│  │  文档A："如何取消订单"       → 向量A2：[0.21, 0.79, ...]   │        │
│  │  文档B："订单怎么退掉"       → 向量A3：[0.19, 0.81, ...]   │        │
│  │  文档C："不想买了怎么办"   → 向量A4：[0.18, 0.82, ...]   │        │
│  │                                                          │        │
│  │  计算相似度：                                              │        │
│  │  • 文档A：0.98  ← 最相似 ✓                               │        │
│  │  • 文档B：0.96  ← 很相似 ✓                               │        │
│  │  • 文档C：0.94  ← 相似 ✓                                 │        │
│  │                                                          │        │
│  │  优点：意思相同的内容都能找到！                          │        │
│  │                                                          │        │
│  └─────────────────────────────────────────────────────────┘        │
│                                                                       │
└─────────────────────────────────────────────────────────────────────┘
```

**相似度计算方法：**

```
┌─────────────────────────────────────────────────────────────────────┐
│                        相似度计算方法                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  方法1：余弦相似度（Cosine Similarity）                             │
│  ┌─────────────────────────────────────────────────────────┐        │
│  │                                                          │        │
│  │  公式：                                                  │        │
│  │       A · B           向量A和B的点积                    │        │
│  │  cos = ───────── = ──────────────────────              │        │
│  │       |A| × |B|      两个向量的模相乘                   │        │
│  │                                                          │        │
│  │  取值范围：[-1, 1]                                      │        │
│  │  • 1：完全相同方向                                      │        │
│  │  • 0：垂直（无关联）                                    │        │
│  │  • -1：完全相反                                         │        │
│  │                                                          │        │
│  │  优点：只关心方向，不关心大小                            │        │
│  │                                                          │        │
│  └─────────────────────────────────────────────────────────┘        │
│                                                                       │
│  方法2：点积（Dot Product）                                         │
│  ┌─────────────────────────────────────────────────────────┐        │
│  │                                                          │        │
│  │  公式：A · B = Σ(Ai × Bi)                              │        │
│  │        = A1×B1 + A2×B2 + ... + An×Bn                   │        │
│  │                                                          │        │
│  │  取值范围：[-∞, +∞]                                     │        │
│  │  • 值越大：越相似                                       │        │
│  │                                                          │        │
│  │  优点：计算简单，速度快                                  │        │
│  │  缺点：向量大小会影响结果                                │        │
│  │                                                          │        │
│  └─────────────────────────────────────────────────────────┘        │
│                                                                       │
│  方法3：欧氏距离（Euclidean Distance）                              │
│  ┌─────────────────────────────────────────────────────────┐        │
│  │                                                          │        │
│  │  公式：                                                  │        │
│  │  d = √(Σ(Ai - Bi)²)                                    │        │
│  │    = √((A1-B1)² + (A2-B2)² + ...)                      │        │
│  │                                                          │        │
│  │  取值范围：[0, +∞]                                      │        │
│  │  • 0：完全相同                                          │        │
│  │  • 值越大：越不相似                                     │        │
│  │                                                          │        │
│  │  优点：物理意义直观                                      │        │
│  │  缺点：高维空间效果不如余弦相似度                        │        │
│  │                                                          │        │
│  └─────────────────────────────────────────────────────────┘        │
│                                                                       │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.3.2 RAG引擎核心代码

**文件位置：** [chat-service/rag.py](../../chat-service/rag.py)

```python
# RAG引擎 - 混合检索实现
# 文件：backend/chat-service/rag.py

class RAGEngine:
    """
    RAG引擎，支持混合检索
    
    设计思路：
    1. FAQ优先：简单问题用FAQ，节省成本
    2. RAG兜底：复杂问题用向量检索，保证准确性
    3. 可配置阈值：根据业务调整匹配严格度
    """
    
    def __init__(self):
        # 知识服务地址（微服务通信）
        self.knowledge_service_url = settings.KNOWLEDGE_SERVICE_URL
        
        # FAQ匹配阈值（高于此值直接返回FAQ答案）
        self.faq_threshold = 0.85
        
        # RAG检索阈值（低于此值可能需要特殊处理）
        self.rag_threshold = 0.7
    
    async def retrieve(
        self,
        enterprise_id: str,
        query: str,
        knowledge_base_ids: Optional[List[str]] = None,
        top_k: int = None,
        enable_rerank: bool = True,
        use_cross_encoder: bool = True
    ) -> Dict[str, Any]:
        """
        混合检索主入口
        
        Args:
            enterprise_id: 企业ID（多租户）
            query: 用户问题
            knowledge_base_ids: 指定知识库ID列表（可选）
            top_k: 返回Top-K个结果
            enable_rerank: 是否启用重排序
            use_cross_encoder: 是否使用Cross-Encoder精排
        
        Returns:
            {
                "source_type": "faq" | "rag",
                "answer": str,
                "sources": List[Dict],
                "chunks": List[Dict]  # RAG情况下的详细片段
            }
        """
        
        # Step 1: FAQ精确匹配
        faq_result = await self._faq_match(enterprise_id, query)
        if faq_result and faq_result["score"] >= self.faq_threshold:
            # FAQ匹配度高，直接返回
            return {
                "source_type": "faq",
                "answer": faq_result["answer"],
                "sources": [faq_result],
                "chunks": []
            }
        
        # Step 2: FAQ匹配度不够，进入RAG检索
        rag_result = await self._rag_retrieve(
            enterprise_id,
            query,
            knowledge_base_ids,
            top_k or 5,
            enable_rerank,
            use_cross_encoder
        )
        
        return rag_result
    
    async def _faq_match(
        self,
        enterprise_id: str,
        query: str
    ) -> Optional[Dict[str, Any]]:
        """
        FAQ精确匹配
        
        实现思路：
        1. 在FAQ表中进行关键词匹配
        2. 同时使用语义相似度
        3. 综合评分，返回最佳匹配
        """
        
        # 调用Knowledge Service的FAQ匹配API
        async with httpx.AsyncClient() as client:
            response = await client.post(
                f"{self.knowledge_service_url}/faq/match",
                json={
                    "query": query,
                    "enterprise_id": enterprise_id,
                    "threshold": self.faq_threshold
                },
                timeout=5.0
            )
        
        if response.status_code == 200:
            result = response.json()
            if result.get("matched"):
                return {
                    "type": "faq",
                    "answer": result["answer"],
                    "question": result["question"],
                    "score": result["score"],
                    "source": "FAQ知识库"
                }
        
        return None
    
    async def _rag_retrieve(
        self,
        enterprise_id: str,
        query: str,
        knowledge_base_ids: Optional[List[str]],
        top_k: int,
        enable_rerank: bool,
        use_cross_encoder: bool
    ) -> Dict[str, Any]:
        """
        RAG向量检索
        
        实现思路：
        1. 调用Knowledge Service获取向量嵌入
        2. 在ChromaDB中检索相似文档
        3. 可选：使用Cross-Encoder重排序
        4. 构造上下文返回
        """
        
        # Step 2.1: 获取检索结果
        async with httpx.AsyncClient() as client:
            response = await client.post(
                f"{self.knowledge_service_url}/retrieve",
                json={
                    "query": query,
                    "enterprise_id": enterprise_id,
                    "knowledge_base_ids": knowledge_base_ids,
                    "top_k": top_k,
                    "enable_rerank": enable_rerank,
                    "use_cross_encoder": use_cross_encoder
                },
                timeout=10.0
            )
        
        result = response.json()
        
        # Step 2.2: 构造上下文
        context = self._build_context(result["chunks"])
        
        return {
            "source_type": "rag",
            "answer": None,  # 需要LLM生成
            "context": context,
            "sources": result["chunks"],
            "chunks": result["chunks"]
        }
    
    def _build_context(self, chunks: List[Dict]) -> str:
        """
        构造RAG上下文
        
        把检索到的文档片段构造成LLM可以理解的格式
        """
        if not chunks:
            return ""
        
        context_parts = []
        context_parts.append("参考知识库内容：\n")
        
        for i, chunk in enumerate(chunks, 1):
            context_parts.append(
                f"[{i}] 来源：{chunk.get('source', '未知')}\n"
                f"内容：{chunk.get('text', '')}\n"
            )
        
        return "\n".join(context_parts)
```

```python
# 文档分块器
# 文件：backend/knowledge-service/chunker.py

class TextChunker:
    """
    文本分块器
    
    支持多种分块策略：
    1. 固定长度分块（简单快速）
    2. 句子级别分块（语义较好）
    3. 段落级别分块（推荐）
    """
    
    def __init__(
        self,
        chunk_size: int = 500,
        overlap: int = 50,
        strategy: str = "paragraph"
    ):
        """
        初始化分块器
        
        Args:
            chunk_size: 每个块的最大字符数
            overlap: 块之间的重叠字符数（防止上下文丢失）
            strategy: 分块策略 ("fixed", "sentence", "paragraph")
        """
        self.chunk_size = chunk_size
        self.overlap = overlap
        self.strategy = strategy
    
    def chunk(self, text: str, metadata: Dict = None) -> List[Dict]:
        """
        将文本分割成块
        
        Args:
            text: 待分割的文本
            metadata: 文档元数据（来源、标题等）
        
        Returns:
            List[Dict]: 块列表，每个块包含text和metadata
        """
        if self.strategy == "fixed":
            return self._chunk_fixed(text, metadata)
        elif self.strategy == "sentence":
            return self._chunk_by_sentence(text, metadata)
        elif self.strategy == "paragraph":
            return self._chunk_by_paragraph(text, metadata)
        else:
            raise ValueError(f"Unknown chunking strategy: {self.strategy}")
    
    def _chunk_fixed(self, text: str, metadata: Dict) -> List[Dict]:
        """
        固定长度分块
        
        简单但可能打断句子
        """
        chunks = []
        start = 0
        
        while start < len(text):
            end = start + self.chunk_size
            chunk_text = text[start:end]
            
            chunks.append({
                "text": chunk_text,
                "metadata": {
                    **(metadata or {}),
                    "start_char": start,
                    "end_char": end
                }
            })
            
            # 滑动窗口：移动 chunk_size - overlap
            start += self.chunk_size - self.overlap
        
        return chunks
    
    def _chunk_by_sentence(self, text: str, metadata: Dict) -> List[Dict]:
        """
        按句子分块
        
        保持句子完整性，但块大小不均匀
        """
        # 分割句子（简单的中文句子分割）
        import re
        sentences = re.split(r'[。！？\n]+', text)
        
        chunks = []
        current_chunk = ""
        start = 0
        
        for sentence in sentences:
            sentence = sentence.strip()
            if not sentence:
                continue
            
            # 如果加上这个句子会超过chunk_size
            if len(current_chunk) + len(sentence) > self.chunk_size:
                # 保存当前块
                if current_chunk:
                    chunks.append({
                        "text": current_chunk,
                        "metadata": {
                            **(metadata or {}),
                            "start_char": start
                        }
                    })
                
                # 开始新块（使用overlap保持连续性）
                if self.overlap > 0 and current_chunk:
                    # 取最后一个overlap字符作为新块开头
                    current_chunk = current_chunk[-self.overlap:]
                    start = start + len(current_chunk) - self.overlap
                else:
                    current_chunk = ""
                    start = start + len(sentence)
            
            current_chunk += sentence + "。"
        
        # 保存最后一个块
        if current_chunk.strip():
            chunks.append({
                "text": current_chunk,
                "metadata": {
                    **(metadata or {}),
                    "start_char": start
                }
            })
        
        return chunks
    
    def _chunk_by_paragraph(self, text: str, metadata: Dict) -> List[Dict]:
        """
        按段落分块（推荐）
        
        保持段落结构，语义完整性好
        """
        # 按换行符分割段落
        paragraphs = text.split('\n')
        
        chunks = []
        current_chunk = ""
        start = 0
        
        for para in paragraphs:
            para = para.strip()
            if not para:
                continue
            
            # 如果加上这个段落会超过chunk_size
            if len(current_chunk) + len(para) > self.chunk_size:
                # 保存当前块
                if current_chunk:
                    chunks.append({
                        "text": current_chunk,
                        "metadata": {
                            **(metadata or {}),
                            "start_char": start
                        }
                    })
                
                # 开始新块
                current_chunk = para + "\n"
                start += len(current_chunk)
            else:
                current_chunk += para + "\n"
        
        # 保存最后一个块
        if current_chunk.strip():
            chunks.append({
                "text": current_chunk,
                "metadata": {
                    **(metadata or {}),
                    "start_char": start
                }
            })
        
        return chunks
```

# 向量化服务
# 文件：backend/knowledge-service/embeddings.py

class EmbeddingService:
    """
    Embedding服务
    
    支持多种Embedding模型：
    1. OpenAI (text-embedding-ada-002)
    2. 本地模型 (M3E, BGE)
    """
    
    def __init__(
        self,
        model_type: str = "openai",
        model_name: str = "text-embedding-ada-002"
    ):
        self.model_type = model_type
        self.model_name = model_name
        
        # 根据模型类型初始化
        if model_type == "openai":
            self.client = OpenAI(api_key=settings.OPENAI_API_KEY)
        elif model_type in ["m3e", "bge"]:
            # 本地模型初始化
            from sentence_transformers import SentenceTransformer
            self.model = SentenceTransformer(model_name)
    
    async def embed_text(self, text: str) -> List[float]:
        """
        将单个文本转换为向量
        
        Args:
            text: 输入文本
        
        Returns:
            List[float]: 文本向量
        """
        if self.model_type == "openai":
            response = self.client.embeddings.create(
                model=self.model_name,
                input=text
            )
            return response.data[0].embedding
        else:
            # 本地模型
            embedding = self.model.encode(text)
            return embedding.tolist()
    
    async def embed_batch(
        self,
        texts: List[str],
        batch_size: int = 32
    ) -> List[List[float]]:
        """
        批量将文本转换为向量
        
        Args:
            texts: 文本列表
            batch_size: 批处理大小
        
        Returns:
            List[List[float]]: 向量列表
        """
        all_embeddings = []
        
        # 分批处理
        for i in range(0, len(texts), batch_size):
            batch = texts[i:i + batch_size]
            
            if self.model_type == "openai":
                response = self.client.embeddings.create(
                    model=self.model_name,
                    input=batch
                )
                embeddings = [item.embedding for item in response.data]
            else:
                embeddings = self.model.encode(batch).tolist()
            
            all_embeddings.extend(embeddings)
        
        return all_embeddings

### 2.6.2 重排序实现

```python
# 重排序服务
# 文件：backend/knowledge-service/rerank.py

class Reranker:
    """
    文档重排序器
    
    使用Cross-Encoder模型对初步检索结果进行精排
    """
    
    def __init__(self, model_name: str = "cross-encoder/ms-marco"):
        """
        初始化重排序器
        
        Args:
            model_name: Cross-Encoder模型名称
        """
        from sentence_transformers import CrossEncoder
        
        self.model = CrossEncoder(model_name)
    
    async def rerank(
        self,
        query: str,
        documents: List[str],
        top_k: int = 5
    ) -> List[Dict]:
        """
        对文档进行重排序
        
        Args:
            query: 查询问题
            documents: 候选文档列表
            top_k: 返回Top-K个
        
        Returns:
            重排序后的文档列表，包含相关度分数
        """
        # 构造(query, document)对
        pairs = [(query, doc) for doc in documents]
        
        # 批量预测相关度分数
        scores = self.model.predict(pairs)
        
        # 按分数排序
        doc_scores = list(zip(documents, scores))
        doc_scores.sort(key=lambda x: x[1], reverse=True)
        
        # 返回Top-K
        results = []
        for i, (doc, score) in enumerate(doc_scores[:top_k]):
            results.append({
                "text": doc,
                "rerank_score": float(score),
                "rank": i + 1
            })
        
        return results
```
