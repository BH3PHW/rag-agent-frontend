## 23.2 后端启动

### 步骤1：克隆项目（如果还没有）

```bash
cd /your/workspace
# 假设项目已经在 /workspace 目录下
```

### 步骤2：安装Python依赖

每个服务都有自己的依赖，需要单独安装：

```bash
# 1. API Gateway
cd /workspace/backend/api-gateway
pip install -r requirements.txt

# 2. User Service
cd /workspace/backend/user-service
pip install -r requirements.txt

# 3. Chat Service
cd /workspace/backend/chat-service
pip install -r requirements.txt

# 4. Knowledge Service
cd /workspace/backend/knowledge-service
pip install -r requirements.txt

# 其他服务（可选）
# admin-service, alert-service, analytics-service, channel-service
```

### 步骤3：配置数据库

创建PostgreSQL数据库：

```sql
-- 连接到PostgreSQL
psql -U postgres

-- 创建数据库
CREATE DATABASE rag_kb;
CREATE DATABASE rag_chat;
CREATE DATABASE rag_user;

-- 创建用户（可选）
CREATE USER rag_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE rag_kb TO rag_user;
```

更新配置文件（每个服务的 config.py 或 .env）：

```python
# 示例：user-service/config.py
DATABASE_URL = "postgresql://postgres:password@localhost:5432/rag_user"
REDIS_URL = "redis://localhost:6379/0"
```

### 步骤4：启动服务（需要多个终端）

**终端1 - API Gateway**
```bash
cd /workspace/backend/api-gateway
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

**终端2 - User Service**
```bash
cd /workspace/backend/user-service
uvicorn main:app --reload --host 0.0.0.0 --port 8001
```

**终端3 - Chat Service**
```bash
cd /workspace/backend/chat-service
uvicorn main:app --reload --host 0.0.0.0 --port 8002
```

**终端4 - Knowledge Service**
```bash
cd /workspace/backend/knowledge-service
uvicorn main:app --reload --host 0.0.0.0 --port 8003
```

### 步骤5：验证后端

打开浏览器访问API文档：
- API Gateway: http://localhost:8000/docs
- Chat Service: http://localhost:8002/docs
- Knowledge Service: http://localhost:8003/docs

看到Swagger UI说明启动成功！

## 23.5 常见问题

### 问题1：端口被占用

```bash
# Linux/Mac: 查看谁占用了端口
lsof -i :8000

# 或杀掉占用进程
kill -9 <PID>

# Windows:
netstat -ano | findstr :8000
taskkill /PID <PID> /F
```

### 问题2：数据库连接失败

- 检查PostgreSQL是否在运行
- 检查用户名/密码是否正确
- 检查数据库是否已创建
- 查看 config.py 中的数据库连接字符串

### 问题3：Redis连接失败

- 检查Redis是否在运行：`redis-cli ping` 应该返回 PONG
- 检查Redis地址配置

### 问题4：前端无法连接后端

- 确认后端服务正在运行
- 检查前端 API 配置（通常在 .env 或 config 文件中）
- 打开浏览器开发者工具，查看 Network 标签

## 23.8 使用Docker快速启动（可选）

如果安装了Docker，可以一键启动：

